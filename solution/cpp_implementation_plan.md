# C++ 实现方案拆解（布局求解题）

## 1. 问题定位

这道题本质上是一个带约束的二维矩形布局问题：

- 将 K 个矩形物体摆放到一个给定轮廓内
- 满足不越界、不重叠、不遮挡门
- 物体只能旋转 0 度或 90 度
- 冰箱需要额外考虑开门边的保留空间
- 在可行的前提下，优先选择贴墙摆放

因此，这道题最适合用“几何建模 + 候选位置生成 + 回溯搜索”的方式实现，而不是直接暴力在平面上连续搜索。

---

## 2. 总体实现策略

推荐采用四层结构：

1. **输入解析层**
   - 读取房间轮廓、门、物体列表
   - 转换为统一的数据结构

2. **几何建模层**
   - 点、线段、矩形、多边形
   - 矩形与轮廓、门、其他物体之间的几何关系判断

3. **候选位置生成层**
   - 优先生成贴墙候选
   - 再生成内部网格候选作为兜底

4. **搜索求解层**
   - 按优先级依次放置物体
   - 用 DFS / 回溯找到一组满足约束的方案

这四层拆开后，结构清晰，便于调试，也更适合在 README 中说明算法逻辑。

---

## 3. 建模假设（第一版建议）

为了保证实现稳、代码易写，第一版建议采用以下假设：

- 房间轮廓按正交多边形处理（边平行于 x 轴或 y 轴）
- 门为水平或竖直门
- 所有物体均为轴对齐矩形
- 物体旋转仅允许 0 度和 90 度
- 物体之间允许边贴边，但不允许面积重叠

这些假设非常适合这道题，也能显著降低实现难度。

## 5. 核心数据结构设计

### 5.1 基础几何结构

建议定义以下结构：

- `Point`：表示二维点 `(x, y)`
- `Segment`：表示线段，由两个点组成
- `Rect`：表示轴对齐矩形，用 `(x1, y1, x2, y2)` 表示

矩形统一使用 AABB（axis-aligned bounding box）表示，这样：

- 碰撞检测简单
- 贴墙逻辑简单
- 和 0 / 90 度旋转天然匹配

### 5.2 题目对象结构

建议定义：

- `Door`：包含门线段和门类型（内开 / 外开）
- `Item`：包含名称、类型、长、宽
- `Placement`：表示一个物体的摆放结果
  - 所属物体编号
  - 中心点
  - 旋转角度
  - 本体矩形
  - 若为冰箱，还包含开门保留区
- `Problem`：表示整个输入
- `Solution`：表示输出结果

这样建模后，输入、求解、输出都能统一使用这些结构。

## 6. 几何模块要完成什么

几何模块是整个程序的基础，优先实现。

### 6.1 矩形工具函数

至少需要：

- 根据中心点 + 宽高生成矩形
- 计算矩形宽高
- 获取矩形四个顶点
- 判断两个矩形是否重叠

其中“是否重叠”建议按以下规则：

- 边贴边不算冲突
- 只有面积相交才算冲突

### 6.2 点与多边形关系

至少需要：

- 判断点是否在线段上
- 判断点是否在多边形内

建议：

- 边界视为“在内部”
- 可用射线法判断点是否在多边形内

### 6.3 矩形是否在轮廓内

这是关键判断函数之一。

建议按三步判断：

1. 矩形四个角都在轮廓内或边界上
2. 矩形中心点也在轮廓内（对凹多边形更稳）
3. 矩形边与轮廓边不能发生“真正穿出”的相交

这样可以避免“角点在内，但矩形边穿出凹区域”的误判。

## 7. 门约束的实现思路

门需要建模成禁放区。

### 7.1 外开门

外开门对室内影响较小，第一版可简化为：

- 将门洞线段附近的一条窄条区域视为禁放区

### 7.2 内开门

内开门需要占用一个额外区域。

建议处理方式：

- 先计算门宽 `N`
- 在门所在边朝室内方向扩展出一个 `N x N` 的矩形禁区

为便于实现，第一版可用“房间包围盒中心相对门的位置”来近似判断室内方向：

- 若门是水平门，则判断禁区向上还是向下扩展
- 若门是竖直门，则判断禁区向左还是向右扩展

之后统一将该区域视为不可摆放区。

## 8. 特殊物体规则实现

### 8.1 冰箱（FRIDGE）

冰箱除了本体矩形，还必须有一个“开门边保留区”。

推荐统一约定：

- `angle = 0` 时，上侧长边为开门边
- `angle = 90` 时，右侧长边为开门边

然后根据本体矩形，在对应一侧额外扩一块条状禁区：

- 该区域不允许其他物体进入
- 该区域也必须落在房间轮廓内

这可以直接实现“冰箱开门边前方不能放东西”的要求。

### 8.2 货架（SHELF）

货架没有额外开门区。

只检查本体矩形是否合法即可。

### 8.3 其他类型

`OVERSHELF`、`ICEMAKER` 第一版按普通矩形处理即可。

## 9. 候选位置生成策略

这是算法效率的关键。

不要在整个平面连续搜索，而是先生成一批“有意义的候选位置”。

### 9.1 贴墙候选（优先）

因为题目要求优先贴墙，第一阶段应优先搜索贴墙候选。

对每个物体：

1. 枚举两种角度：0 / 90
2. 计算旋转后的宽高
3. 尝试贴上边、下边、左边、右边
4. 在每条边上生成几个离散中心点，例如：
   - 左端 / 下端
   - 中间
   - 右端 / 上端

这样可快速得到一组贴墙候选位置。

### 9.2 内部候选（兜底）

若纯贴墙解无解，再生成内部候选。

建议做法：

1. 求房间的包围盒
2. 在包围盒内按固定网格步长扫描
3. 对每个格点尝试 0 / 90 两种角度
4. 通过基础合法性检查后加入候选池

网格步长建议：

- 第一版用 `1.0`
- 如果需要更细结果可改成 `0.5`

---

## 10. 合法性检查逻辑

一个摆放方案是否合法，需要检查以下几项。

### 10.1 本体矩形在轮廓内

调用“矩形是否在轮廓内”的几何函数。

### 10.2 本体不与门禁区冲突

物体本体不能和门禁放区相交。

### 10.3 冰箱开门区合法

若当前物体是冰箱：

- 开门保留区也必须在轮廓内
- 开门保留区不能压到门禁区

### 10.4 不与已放置物体重叠

- 当前物体本体不能与已放物体本体重叠
- 当前物体不能进入已放冰箱的开门保留区
- 若当前物体是冰箱，则其开门保留区也不能压到已放物体

只有全部通过，才认为该位置可摆放。

## 11. 搜索求解层（DFS / 回溯）

在有了候选位置后，使用 DFS / 回溯搜索一组可行解。

### 11.1 搜索顺序优化

为了减少搜索量，不要按原输入顺序直接摆放。

建议按以下优先级排序：

1. 冰箱优先（约束多）
2. 候选数量少的优先（更难摆）
3. 面积大的优先

这样可以显著降低回溯深度和分支数。

### 11.2 DFS 递归逻辑

递归过程：

1. 取当前要摆放的物体
2. 遍历其所有候选位置
3. 若候选合法，则加入已摆放列表
4. 递归摆放下一个物体
5. 若后续失败，则撤销当前摆放并尝试下一个候选
6. 全部物体摆完，则找到可行解

这是最稳妥、最容易调试的求解方式。

---

## 12. “优先贴墙”如何体现

这不是硬约束，而是偏好。

推荐用**两阶段求解**来实现：

### 阶段一：仅使用贴墙候选

- 所有物体只使用贴墙候选位置
- 若能找到解，直接返回

### 阶段二：贴墙候选 + 内部候选

- 若阶段一失败，再为物体补充内部候选
- 重新进行 DFS 搜索

这样实现简单、清晰，而且与题意完全一致。

## 13. 输入输出设计建议

### 13.1 输入格式

建议采用简单文本格式，例如：

```text
CONTOUR 4
0 0
10 0
10 8
0 8

DOOR
4 0 6 0 INWARD

ITEMS 4
item1 FRIDGE 2 1
item2 SHELF 3 1
item3 OVERSHELF 2 1
item4 ICEMAKER 1 1
```

优点：

- 易读
- 易手写测试
- 易写解析器

### 13.2 输出格式

建议输出：

```text
FEASIBLE YES
item1 center=(1.000, 7.500) angle=0
item2 center=(8.500, 7.500) angle=0
item3 center=(3.000, 1.000) angle=90
item4 center=(9.000, 1.000) angle=0
```

若失败则输出：

```text
FEASIBLE NO
```

这样正好对应题目要求。

## 14. 建议的开发顺序

实现时建议严格按下面顺序推进：

### 第一步：先完成几何基础模块

优先写：

- 点、线段、矩形
- 矩形重叠判断
- 点是否在多边形内
- 矩形是否在轮廓内

因为后面所有逻辑都依赖这些基础能力。

### 第二步：完成门禁区与冰箱保留区

先把门和冰箱的特殊约束建模出来。

### 第三步：完成候选生成

先做贴墙候选，再补内部候选。

### 第四步：完成 DFS 搜索

将前面的模块串起来，得到完整求解器。

### 第五步：补充样例与调试输出

建议增加多个测试样例：

- 简单矩形房间
- 含内开门
- 冰箱靠墙但开门区越界
- 多物体贴边排布
- 无解情况

## 15. 第一版代码目标

第一版代码不必追求“最优布局”，只要做到：

- 能正确判断可行 / 不可行
- 能输出一组合法方案
- 能体现优先贴墙的求解顺序

这已经足以满足题目实现要求，也方便后续再继续优化。

---

## 16. 后续可优化方向

在第一版跑通后，可以继续升级：

1. **更精确的贴墙候选生成**
   - 不再只基于房间包围盒
   - 改为逐条真实轮廓边生成候选

2. **更细粒度的内部搜索**
   - 将网格步长从 `1.0` 调小到 `0.5` 或 `0.25`

3. **增加布局评分函数**
   - 与墙接触越多分越高
   - 越靠近墙分越高
   - 靠近门或浪费空间可扣分

4. **支持更复杂轮廓**
   - 针对凹多边形做更稳健的候选生成

5. **支持更完整的门方向判断**
   - 用轮廓方向和法向量准确判断门朝室内的方向

---

## 17. 结论

这道题最合适的实现路径是：

- 用统一的数据结构表示房间、门、物体和摆放结果
- 用几何函数负责合法性判断
- 用候选生成降低搜索空间
- 用 DFS / 回溯完成可行解搜索
- 用“两阶段求解”自然体现优先贴墙

这套方案的优点是：

- 易于实现
- 易于调试
- 易于讲解
- 易于继续优化

非常适合作为 C++ 工程化实现的第一版。
